Funzionalità del progetto

Il seguente programma, come da specifiche di progetto, scarica i dataset da file .csv più grande, effettua il parsing dei dati e metadati ed implementa la loro serializzazione. Il programma si occupa anche di implementare le risposte alle richieste dell’utente (ritorno dei dati, metadati, statistiche con relativi filtraggi).

Download e configurazione

Per far partire il progetto basta scaricarlo da gitHub ed aprirlo su Eclipse.
Il programma richiede l’uso di framework e librerie esterne, quindi per far sì che compili bisogna aggiungere nel ClassPath il file
json-simple-1.1.1.jar, già presente nella directory del progetto.

Sintassi per richieste REST API

L’utente per poter utilizzare le funzionalità del programma, deve eseguire l’applicazione ed aprire il browser WEB. Le richieste verranno scritte come URL con la seguente sintassi:

http://localhost:8080/“tipoRichiesta”

All’interno della query string, la string tipoRichiesta serve per indicare il tipo di informazione che si vuole richiedere, tra le seguenti opzioni:

  • dati [Ritorna la lista in formato JSON dei Bean contenuti nel data-set];
  • metadati [Ritorna la lista in formato JSON dei metadati relativi agli attributi];
  • stats [Ritorna le statistiche in formato JSON relative ad un attributo(varia a seconda del tipo)].

Per poter ricevere statistiche relative ad un attributo bisogna specificarlo con la seguente sintassi:

http://localhost:8080/stats?attributo=“nomeAttributo”

in cui nomeAttributo è il nome dell’attributo.
esempio:

http://localhost:8080/stats?attributo=tipoGara

Per la richiesta di dati e statistiche, è possibile usufruire del filtraggio dati, per poter restituire dati con le proprie specifiche. I tipi di filtri disponibili sono i seguenti:

  • $eq : questo filtro permette di selezionare oggetti con attributo specificato con valore uguale a quello richiesto;
  • $not : questo filtro permette di selezionare oggetti con attributo specificato con valore diverso da quello richiesto;
  • $or : questo filtro permette di selezionare oggetti con attributo specificato uguale a quelli richiesti (l’utente dovrà, in questo caso, inserire due valori);
  • $mag : questo filtro permette di selezionare oggetti con attributo numerico specificato con valore maggiore di quello richiesto;
  • $min : questo filtro permette di selezionare oggetti con attributo numerico specificato con valore minore di quello richiesto;

Nella query string, quindi, l’utente dovrà specificare vari parametri (di cui l’ordine non è importante) a seconda del filtro utilizzato (se si utilizzano filtri per attributi di tipo numerico, non si possono selezonare attributi di tipo String ecc…).
Un esempio di richiesta dati con filtro “$eq” è:

http://localhost:8080/dati?attributo=npgAttoAutorizzativo&filter=$eq&val1=356706

Nel precedente esempio, la richiesta dati è stata filtrata cercando solo oggetti con attributo npgAttoAutorizzativo uguale al valore 356706.

Nota: L’attributo val2 viene utilizzato esclusivamente nel caso del filtro $or che richiede due valori. Tutti gli altri filtri adoperano solo ed esclusivamente l’attributo val1.
